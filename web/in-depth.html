Detailed Data Information

We are using the Justice Centered and Case Centered versions of the SCDB, organized by Issue/Legal Provision. The former has around 100,000 lines of data (1 for each justice's vote in each case) and the latter around 10,000 lines of data (1 for each case). The justice-centered dataset includes all of the information in the case-centered dataset, in addition to information about each justice's votes.

There are 54 or 63 attributes in total for the case-centered and justice-centered databases, respectively. In particular, we are interested in variables which describe the timeline of the court, decision and voting information, and the issue area. These are described below. Information about every attribute, including the specifics of how they are determined, can be found in the SCDB Codebook.

Both databases:

  • term: Term (year) of the court
  • chief: Last name of the Chief Custice of the court
  • issueArea: Number representing which of 14 issue areas the case lies within
  • decisionDirection: 1 if the decision in the case was conservative, 2 if liberal, or 3 if unspecifiable
  • majVotes: the number of votes in the majority

Justice-centered only:

  • justiceName: The (abbreviated) name of each justice
  • direction: 1 if the direction of the justice's vote was conservative, 2 if liberal, or 3 if unspecifiable
  • majority: 2 if the justice voted with the majority, 3 if with the dissent
  • median: 1 if the justice is the median justice, 0 if not

For example, the relevant attributes in a line of data from the justice-centered database might look like this:

justiceName term chief issueArea decisionDirection majVotes direction majority median
RBGinsburg 2015 Roberts 3 2 5 2 2 0

Or, in plain english: In 2015, under Chief Justice Roberts, the court released a liberal decision in a case about First Amendment issues. Justice Ginsburg voted with the majority (liberal). She was not the median justice at the time.



Additions to the SCDB

Some of those are not present in the original dataset, but were added by Professor Sorenson. The newly-introduced variables are the following:

In the both databases:

  • term: explained above
  • medianJJ: The name of the median (or swing-vote) justice
  • lib_median: 1 when the court is liberal and the justice is liberal (when the justice is the swing vote)
  • lib_court: 1 if the court is liberal, 0 if not
  • NOTE: medianJJ and lib_court are all the same for a given term, while the other variables differ for each case, for each justice.

In only the justice-centered database:

  • liberal: 1 if the justice is liberal, 0 if not)
  • median: explained above
